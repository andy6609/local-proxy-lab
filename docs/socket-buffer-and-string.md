# 소켓 데이터는 왜 char 배열로 받나? (HTTP / buffer / char\* 정리)

> 멀티스레드 프록시(`proxy_multithread.cpp`)의 `recv` 버퍼와 클라이언트 `request`를
> 보다가 생긴 의문 정리. 핵심: **"HTTP는 문자열"이 아니라 "소켓은 바이트, char는 1바이트"** 다.

---

## 1. 진실: 소켓은 "문자열"이 아니라 "바이트"를 주고받는다

네트워크로 오가는 건 전부 **바이트(byte) 덩어리**다. 문자열이라서가 아니다.

C/C++에서 **`char` = 정확히 1바이트**. 그래서:

```cpp
char buffer[4096];  // = "1바이트짜리 4096개" = 4096바이트 그릇
```

즉 `char buffer[]`는 "문자 배열"이라기보다 **"받은 바이트를 담을 그릇"**이다.
`recv`가 네트워크에서 바이트를 긁어와 이 그릇에 부어넣는다.
바이트를 담는 표준 도구가 `char`라서 쓰는 것뿐.

---

## 2. 그럼 왜 HTTP는 "문자열처럼" 보이나?

**HTTP/1.1 헤더가 마침 사람이 읽을 수 있는 ASCII 텍스트로 약속돼 있어서** 그렇다.

```
GET / HTTP/1.1
Host: localhost
Connection: close
```

이게 그냥 글자(텍스트)다. 그래서 `printf("%s", buffer)`로 출력하면 사람이 읽게 잘 나온다.

### ⚠️ 하지만 "HTTP = 무조건 문자열"은 아니다

| 부분 | 형식 | %s 출력 |
|---|---|---|
| **헤더(header)** | 텍스트(ASCII) | 잘 보임 ✅ |
| **본문(body)** | 이미지·동영상 등 **바이너리**일 수 있음 | 깨짐 ❌ |

지금은 헤더만 보는 단계라 텍스트로 취급해도 된다.
정확히는 **"바이트를 받아서, 텍스트인 부분을 텍스트로 본다"**가 맞다.

---

## 3. `%s`로 출력하려면 null 종료가 필요 → 두 가지 처리

```cpp
memset(buffer, 0, sizeof(buffer));               // 1. 0으로 싹 채워두고
recv(clientSock, buffer, sizeof(buffer) - 1, 0); // 2. 끝 1바이트는 안 받음
```

- `recv`는 **그냥 바이트만 채우지, 끝에 `\0`(널문자)을 안 붙여준다.**
- `%s`는 "`\0` 나올 때까지 출력"하는데, `\0`이 없으면 엉뚱한 메모리까지 읽어서 터진다.
- 그래서 미리 0으로 채우고 + 끝 1바이트 비워둬서 → **마지막에 `\0`이 보장**되게 한 것.

---

## 4. 클라이언트의 `request`는 왜 `const char*` (포인터)?

여기서 **배열 vs 포인터**의 차이가 핵심.

| | `buffer` | `request` |
|---|---|---|
| 타입 | `char buffer[4096]` (배열) | `const char* request` (포인터) |
| 역할 | **받을** 그릇 (내가 채움) | **보낼** 글자를 가리킴 |
| 메모리 | 내가 4096바이트 **소유** | 이미 있는 문자열을 **가리키기만** |
| 쓰기 | 가능 (recv가 채워야 하니까) | 읽기 전용 (const) |

```cpp
const char* request = "GET / HTTP/1.1\r\nHost: localhost\r\nConnection: close\r\n\r\n";
```

- 이 문자열 `"GET..."`은 **프로그램 안에 이미 박혀있는 고정 텍스트(리터럴)**.
- `request`는 그 텍스트의 **시작 주소를 가리키는 포인터**일 뿐.
  새 그릇을 만들 필요 없이, **있는 걸 가리켜서 `send`에 넘긴다.**
- 받을 때(`buffer`)는 빈 그릇이 필요하니 **배열**,
  보낼 때(`request`)는 이미 있는 글자를 가리키기만 하면 되니 **포인터** — 용도가 반대다.

### 왜 `char*`가 아니라 `const char*`?

C++(.cpp)에서는 **문자열 리터럴을 `char*`에 넣으면 에러/경고**가 난다.
(C++ 규칙이 더 엄격해서 "리터럴은 읽기 전용 = const"로 본다.)
단일 버전 `.c`에서는 `char*`로 써도 통과했지만, `.cpp`에서는 `const`를 붙여야 한다.

---

## 한 줄 요약

> 소켓은 **바이트**를 주고받고, `char`가 곧 **1바이트**라
> `char[]`(받는 그릇)·`char*`(보낼 글자 가리킴)를 쓴다.
> HTTP **헤더**가 텍스트라서 문자열처럼 보이는 것뿐, 원래는 바이트.
> **`buffer` = 내가 채우는 배열, `request` = 이미 있는 문자열을 가리키는 포인터.**
