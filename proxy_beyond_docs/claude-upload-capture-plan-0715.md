# 크롬 → Claude 창 파일 업로드 시나리오 (목표: ~7/15)

> Plan A로 "크롬에서 claude.ai에 파일 업로드"를 잡을 수 있는가에 대한 분석.
> 되는 부분과 아직 불확실한 부분을 나눠 정리하고, 실제로 해야 할 실험을 명시한다.
> 목표 기한: **7/15**

---

## 1단계: 전송 프로토콜 — 아마 되지만 함정 하나

크롬 → claude.ai는 보통 **HTTP/2**, 그리고 claude.ai는 Cloudflare 뒤에 있어서 크롬이 **HTTP/3(QUIC)** 를 먼저 시도할 수도 있다.

여기서 우리 프록시(explicit proxy, 브라우저 프록시 설정)를 켜면:

- **QUIC은 UDP**라, 우리 같은 **TCP 기반 HTTP 프록시로는 통과가 안 된다.** 그래서 크롬이 QUIC을 포기하고 **TCP(h2/1.1)로 폴백**한다. → scope 문서의 "C-1(차단→폴백)"이 자연히 일어나는 상황.
- TCP로 오면 이제 **Plan A의 ALPN 다운그레이드**가 작동 → h2를 막고 **HTTP/1.1로 눌러앉힘** → 우리 파서가 복호화 평문을 읽는다.

**→ 여기까지는 된다.** (단, 크롬이 QUIC을 고집하는 특수 상황이면 Plan C 필요)

---

## 2단계: 업로드 "식별" — 진짜 불확실한 지점 ⚠️

Plan A의 업로드 감지는 **`Content-Type: multipart/form-data`를 찾는** 방식이다. curl `-F`로 테스트했을 땐 이게 multipart라 잡혔다(README 검증).

**문제는 claude.ai가 실제로 어떤 방식으로 파일을 올리는지 모른다는 것.** 웹앱마다 업로드 방식이 다르다:

| 업로드 방식 | Plan A가 잡나? |
|---|---|
| `multipart/form-data` (전통적 폼) | ✅ 바로 잡힘 |
| 별도 업로드 엔드포인트 + JSON/base64 | ❌ 현재 감지기가 못 잡음 |
| 바이너리 직접 POST (`application/octet-stream`) | ⚠️ multipart 아니라 놓칠 수 있음 |
| presigned URL로 S3 같은 스토리지에 직접 PUT | ❌ 아예 다른 도메인/방식 |

즉 **"claude.ai가 multipart를 쓰면 Plan A로 잡히고, 다른 방식이면 지금 코드로는 못 잡는다"** 가 정확한 답이다.

## 이건 이미 "열린 질문"으로 문서에 있다

README §5, §6이 정확히 이걸 말하고 있다:

- §5: *"(비암호화 기준) 파일 업로드 식별 리서치 — AI 창 업로드 시 트래픽 모양 조사, 서비스별 업로드 지문표"*
- §6: *"업로드 '100% 식별 가능한가' (TCP 분할로 단서 조각화) — 실제 캡처로 확인 후 파서 설계"*

즉 프로젝트 설계상 이건 **"실제로 캡처해봐야 안다"** 로 이미 열어둔 상태다. ChatGPT/Claude/네이버가 각각 다르게 올릴 테니, **서비스별 지문표**를 만드는 게 다음 할 일로 잡혀 있다.

---

## 그래서 실제로 해야 할 것 (~7/15)

**"Plan A로 claude.ai 업로드가 잡히나?"의 답 = 직접 캡처해서 봐야 한다.** 순서:

1. 크롬 프록시를 우리 프록시로 설정 + rootCA를 OS 신뢰저장소에 등록
2. Plan A 실행 → claude.ai에서 파일 업로드
3. **복호화된 평문 헤더 덤프**(Plan A의 `SHOW_FULL_HEADERS`)를 보고 확인:
   - 어느 도메인/엔드포인트로 가나? (claude.ai? 별도 업로드 서버?)
   - `Content-Type`이 뭐로 오나? (multipart? octet-stream? json?)
   - 파일 내용이 body에 그대로 있나, base64인가, 조각나 있나?
4. 그 결과에 맞춰 감지 로직을 확장 (multipart만 보던 걸 그 방식까지)

---

## 정리

- **복호화(TLS 벗기기)**: ✅ Plan A로 됨 (QUIC은 TCP 폴백되니까)
- **업로드 식별**: ⚠️ **claude가 multipart면 됨, 아니면 실제 캡처 후 파서 확장 필요** — 이게 바로 지금 프로젝트가 서 있는 지점(7주차 씨앗)

정직하게 말하면 "된다/안 된다"를 지금 단정할 수 없고, **claude.ai 업로드를 한 번 캡처해서 트래픽 모양을 보는 게 다음 실험**이다.

> 참고: 맥에서 이 캡처를 하려면 Plan A의 POSIX 포팅이 필요하다(현재 코드는 WinSock 기반 Windows 전용). Hyper-V Windows VM에서 하면 포팅 없이 원본 그대로 재현 가능(README §7 개발환경 결정 참조).
