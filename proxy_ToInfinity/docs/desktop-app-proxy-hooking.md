# 데스크톱 앱(Electron) 프록시 후킹 원리

> 데스크톱 애플리케이션(예: Claude Desktop, Slack, VS Code 등)의 트래픽을 우리가 만든 로컬 프록시(`proxy_ToInfinity`)로 강제 우회시키는 원리와 방법에 대한 설명입니다.

## 1. 최신 데스크톱 앱의 정체 (Electron Framework)
최근 출시되는 대부분의 데스크톱 앱들은 겉보기엔 OS 네이티브 앱(Native App) 같지만, 내부적으로는 **Electron(일렉트론)**이라는 프레임워크를 사용하여 개발됩니다.
Electron은 쉽게 말해 **"주소창과 탭 UI를 제거한 구글 크롬(Chromium) 브라우저"** 엔진 위에 웹 기술(HTML/CSS/JS)로 만든 화면을 띄워주는 기술입니다. 즉, 앱의 심장은 구글 크롬과 완전히 동일합니다.

## 2. 크롬 명령어(Command-line Flags)의 유전
Claude 데스크톱 앱의 통신 엔진이 Chromium이기 때문에, 구글 크롬 브라우저가 지원하는 강력한 네트워크 커맨드라인 옵션들을 Claude 앱도 100% 똑같이 인식하고 처리합니다.
따라서, 브라우저를 프록시에 물릴 때 사용했던 `--proxy-server` 옵션을 앱을 실행할 때 넘겨주면 앱 전체의 트래픽을 프록시로 납치(Hooking)할 수 있습니다.

## 3. Mac 환경에서의 앱 실행 및 인자 전달
Mac 터미널에서는 `open -a` 명령어를 통해 설치된 애플리케이션을 실행할 수 있습니다. 이때 `--args` 플래그를 사용하면, 그 뒤에 따라오는 인자들을 앱 내부의 Chromium 엔진으로 직접 전달할 수 있습니다.

```bash
# Claude 데스크톱 앱을 로컬 프록시(127.0.0.1:18080)에 물리도록 강제 실행
open -a "Claude" --args --proxy-server="http://127.0.0.1:18080"
```

### 실행 흐름
1. `open -a "Claude"`: Mac OS가 Claude 앱 프로세스를 시작합니다.
2. `--args --proxy-server=...`: 시작된 앱의 내장 Chromium 엔진 초기화 시점에 이 인자가 주입됩니다.
3. **결과:** Claude 앱이 서버(예: `claude.ai`)와 통신하기 위해 생성하는 모든 HTTP/1.1 및 HTTP/2 트래픽이 시스템 프록시 설정과 무관하게 `127.0.0.1:18080` 포트로 강제 우회됩니다.
4. 이를 통해 우리는 브라우저뿐만 아니라 **데스크톱 앱에서 발생하는 파일 업로드(DLP 대상) 트래픽까지 완벽하게 동일한 파서(`StandardMultipartParser`)로 분석하고 추출**할 수 있게 됩니다.
